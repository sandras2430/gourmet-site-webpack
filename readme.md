***PAGODA Restaurant***
A single-page application (SPA) for a fictional restaurant, built from scratch with vanilla JavaScript and Webpack to demonstrate a solid understanding of DOM manipulation and module architecture.

***Project Description***
This project simulates a website for a restaurant, allowing users to navigate between different sections ("Home", "Menu", "Contact") through a dynamic interface. The primary goal was to solidify core concepts of modern web development without relying on external JavaScript frameworks.

Key learning objectives include:

ES6 Modules: Organizing code into separate, reusable files.
DOM Manipulation: Creating, modifying, and removing HTML elements with JavaScript.
Webpack: Configuring a bundler to manage dependencies, optimize assets, and serve the project in a development environment.

***Tech Stack***
HTML5: Semantic structure for the content.
CSS3: Styling with Flexbox, CSS variables, and custom fonts (@font-face).
JavaScript (ES6+): Application logic, event handling, and modules.
Webpack: Module bundler for development and production builds.

***Getting Started***
Follow these instructions to get a copy of the project up and running on your local machine.

***Prerequisites***
A modern web browser (Chrome, Firefox, Edge, etc.).
Node.js and npm installed on your system.
1) Installation
Clone the repository:
git clone https://github.com/your-username/saga-restaurant.git
cd saga-restaurant
2) Install dependencies:
npm install
3) Start the development server:
npm start
4) Open your browser and navigate to http://localhost:8080.

***Project Structure***
The project follows a modular structure to keep the code organized and maintainable.
saga-restaurant/
├── dist/                  # Generated by Webpack (do not touch)
├── src/                   # Source code
│   ├── assets/            # Static assets
│   │   ├── fonts/         # Custom fonts
│   │   └── images/        # Images
│   ├── modules/           # JavaScript modules
│   │   ├── footer.js
│   │   ├── homepage.js
│   │   └── menu.js
│   ├── styles.css         # Main stylesheet
│   ├── template.html      # Base HTML template
│   └── index.js           # Application entry point
├── .gitignore
├── package.json
└── webpack.config.js      # Webpack configuration

Important: On Responsiveness
This project is intentionally not responsive and is optimized for desktop viewing.
Application Logic
index.js: Acts as the "brain" of the application. It imports functions from other modules and manages navigation logic using event listeners on the <header> buttons.
modules/homepage.js: Contains the createhomePage() function, which dynamically generates the homepage content.
modules/menu.js: Exports the createMenuPage() function, which builds and displays the menu items.
modules/footer.js: Exports the foothPage() function, which renders the page footer.
template.html: Provides the static HTML structure, including the <header> with navigation and the <div id="content"> which acts as a canvas for the dynamic content.
